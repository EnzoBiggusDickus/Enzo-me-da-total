1° Tema

Introdução ao teste de software

O teste de software é fundamental para o desenvolvimento de sistemas e tem o objetivo de garantir que não haja bugs ou erros.Um teste de sofware envolve a execução de um sistema ou app para identificar falhas e grantir que o produto final seja de qualidade,atendendo aos requisitos especificados e livre de defeitos.

Objetivos

  • 1. Garantir que o software atenda aos requisitos especificados.
  • 2. Detectar falhas ou erros no sistema.
  • 3. Garantir que haja um aboa qualidade para que possa ser utilizado pelos usuários da melhor maneira.
  • 4. Verificar se o sistema é fácil de usar e atende as expectativas de desempenho.

    • Tipos de teste

    Teste Funcional:Verifica se o software funciona de acordo com os requisitos e expectativas.

    Exemplo

  • Teste de desempenho.
  • Teste de segurança.
  • Teste de usuabilidade.

    • Ferramentas de teste

  • JUnit e TestNG: Teste de java.
  • Selenium: Teste de interface.
  • JMeter: Teste de carga e desempenho.
  • SonarQube: Teste de qualidade.
  • Postman: Teste de API.

    • 2° Tema

    Conceito de qualidade de software: Determina se um software atende aos requisitos do usuário e cumpre a função de maneira eficiente.Em outras palavras qualidade,desempenho,defeitos e usabilidade.

    Funcionalidade

    • Capacidade de realizar funções especificas de maneira correta e eficaz.
    • Precisão dos resultados conforme o requisitado.

    Como é medido a qualidade de software

    • Satisfação do usuário
    • Números de defeitos
    • Tempo de resposta
    • Taxa de falha

    Habilidades

      Compreender e aplicar técnicas de qualidade de software.

      Entender o que são testes funcionais,não funcionais e estruturais em sistemas.

      Executar testes funcionais e estrututais em sistemas.

    Benefícios

    • Proporcionar a concorrência justa.
    • Estimular a melhoria contínua da qualidade.
    • Informar e proteger o consumidor.
    • Proteger o mercado interno.
    • Agregar valor as marcas.

    3° Tema

    Ciclos de vida de software

    É o processo de desenvolvimento desde a concepção até a manutenção após implementação o objetivo do (SDLC - Software Development Life Cycle) é garantir que o software seja desenvolvido com alta qualidade.

    Objetivos

  • Planejamento: Requisitos do cliente,definição de metas e análise dos requisitos.
  • Desenvolvimento: Definir estrutura,design e codificar sistemas.
  • Teste e manutenção:Testar problemas e concerta-los

    • Habilidades

    Compreender os métodos de ciclo de um desenvolvimento de software.

    Verificar e validar correspondência entre especificação e produto testado.

    4° Tema

    Processo Unificado da Rational

    É um processo de engenharia de software criado para apoiar o desnevolvimento.O principal objetivo don RUP é ataentender as necessidades dos usuários garantindo uma produção de software del alta qualidade que cumpra um cronograma e um orçamento previsível.O RUP define perfeitamente quem é o responsável pelo que,como as coisas deverão der feitas e quando devem ser realizadas.

    Fases

  • Elaboração: Revisar os riscos,arquitetura,custos e se o projeto é confiável.
  • Construção: Desenvolve ou adquire os componentes de software e codificação.
  • Concepção/Iniciação: Abrange as tarefas de comunicação com o cliente e o planejamento,analise de riscos,estimativas de custos,prazos,prioridades e requisitos do sistema.
  • Transição: Entrega do software,testes,disponibilizar o sistema,facilidade de compreenção,treinamento dos usuários finais e testes da versão beta do sistema.

    • 5° Tema
